This repository follows a phase-based development workflow.

Before starting any work:

1. Read all documentation under /docs.
2. Understand the current implementation.
3. Never break existing functionality.
4. Complete ONLY the requested phase.
5. Test thoroughly.
6. Update CHANGELOG.md.
7. Commit with a professional message.
8. Push to GitHub.
9. Stop and wait for the next phase.

Never skip phases.
Never combine phases.
Always prioritize maintainability, security, scalability, and code quality.
